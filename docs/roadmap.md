# Forward Plan

## Step 1 - Reduce Technical Debt In Build And Runtime

- Постепенно убрать зависимость от `typescript.ignoreBuildErrors` в [`next.config.ts`](../next.config.ts).
- Отдельно локализовать типовые проблемы editor stack и исправлять их пакетно.
- Понять, является ли нестабильность `next dev` в sandbox только проблемой среды или есть дополнительная причина в коде.

## Step 2 - Consolidate Study 3

- Пройтись по `study-3` UX как по единственной версии учебников.
- Уточнить и усилить сценарии:
  - загрузка книги;
  - HTML pipeline;
  - explain;
  - translate;
  - chat;
  - сохранение и кеширование страниц.
- Развивать уже добавленный vocabulary UI внутри study flow:
  - словарь;
  - режим `учить лексику`;
  - мнемотехника как вспомогательный слой внутри lexical cards;
  - дальнейшее усложнение карточек и повторений без смешивания с knowledge cards;
  - edge-case сценарии удаления книг, page-entry и связанных vocabulary cards.

## Step 3 - Harden Planner

- Проверить edge-case сценарии для новых вложенных подзадач:
  - редактирование подзадач в daily plan;
  - удаление родительских задач с вложенным деревом;
  - конкурирующие изменения подзадач между общим и дневным планом.
- Проверить остальные edge-case сценарии дневных планов:
  - перенос задач;
  - конкурирующие редактирования;
  - пересборка дня при изменении источников;
  - уведомления и дедлайны после серии массовых изменений.
- Добавить явные smoke checks для критичных planner flows.

## Step 4 - Clarify Knowledge And Vocabulary Strategy

- Проверить edge-case сценарии после разделения knowledge base на `жизнь` и `работа`:
  - прямые переходы по `cardId` без workspace query;
  - перенос карточек между пространствами, если такой сценарий когда-нибудь появится;
  - видимость тегов и сфер только в рамках выбранного workspace.
- Решение принято: `vocabulary` является подпространством `study-3`, а не частью knowledge cards.
- Следующие шаги:
  - усилить learning steps и scheduling интервальных повторений поверх уже добавленного mnemonic-layer;
  - расширить сценарии карточек для лексики, не перегружая UX;
  - проверить edge-case случаи массового добавления слов из одного учебника;
  - при необходимости уточнить стратегию судьбы vocabulary entries после удаления исходного учебника.

## Step 5 - Improve Project Observability

- Добавить более формализованные проверки post-deploy сценариев.
- При необходимости оформить scripts для типовых production smoke tests как стандартный набор сопровождения.
- Обновить docs после появления устойчивого релизного ритуала.

## Step 6 - Stabilize Automated Tests

- Довести integration и Playwright suites до устойчивого зелёного состояния на CI.
- Расширить browser coverage на planner daily flows, cards workspace flows и study-3 reader edge cases.
- Убрать источники флаки в локальном test runtime и закрепить удобный локальный запуск.
